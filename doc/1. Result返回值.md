# Result返回值

## 说明

>  Result返回值是一个大家在和前端进行交互的时候他是很让大家头疼的，因为如果不是很规范，用起来是非麻烦，但是如果使用`xiaoUtils`中的Result就会变得规范和简易。

## API介绍

### 该类型状态码表示动作被成功接收、理解和接受

```markdown
- `result200(Object message, String path);` ===> 请求被成功地完成，所请求的资源发送到客户端。
- `result201(Object message, String path);` ===> 提示知道新文件的URL。
- `result202(Object message, String path);` ===> 接受并处理，但处理未完成。
- `result203(Object message, String path);` ===> 返回信息不确定或不完整。
- `result204(Object message, String path);` ===> 收到请求，但返回信息为空。
- `result205(Object message, String path);` ===> 服务器完成了请求，用户必须复位当前已经浏览过的文件。
- `result206(Object message, String path);` ===> 服务器已经完成了部分用户的GET请求。
```

### 该类型状态码表示为了完成指定的动作，必须接受进一步处理。

```markdown
- `result300(Object message, String path);` ===> 请求的资源可在多处获得。
- `result301(Object message, String path);` ===> 本网页被永久性转移到另一个URL。
- `result302(Object message, String path);` ===> 请求的网页被重定向到新的地址。
- `result303(Object message, String path);` ===> 建议用户访问其他URL或访问方式。
- `result304(Object message, String path);` ===> 自从上次请求后，请求的网页未修改过。
- `result305(Object message, String path);` ===> 请求的资源必须从服务器指定的地址获得。
- `result306(Object message, String path);` ===> 声明请求的资源临时性删除。
```

### 该类型状态码表示请求包含错误语法或不能正确执行。

```markdown
- `result400(Object message, String path);` ===> 客户端请求有语法错误。
- `result401(Object message, String path);` ===> 请求未经授权。
- `result402(Object message, String path);` ===> 保留有效ChargeTo头响应。
- `result403(Object message, String path);` ===> 禁止访问，服务器收到请求，但拒绝提供服务。
- `result404(Object message, String path);` ===> 可连接服务器，但服务器无法取得所请求的网页，请求资源不存在。
- `result405(Object message, String path);` ===> 用户在Request-Line字段定义的方法不被允许。
- `result406(Object message, String path);` ===> 根据用户发送的Accept，请求资源不可访问。
- `result407(Object message, String path);` ===> 用户必须首先在代理服务器上取得授权。
- `result408(Object message, String path);` ===> 客户端没有在用户指定的时间内完成请求。
- `result409(Object message, String path);` ===> 对当前资源状态，请求不能完成。
- `result410(Object message, String path);` ===> 服务器上不再有此资源。
- `result411(Object message, String path);` ===> 服务器拒绝用户定义的Content-Length属性请求。
- `result412(Object message, String path);` ===> 一个或多个请求头字段在当前请求中错误。
- `result413(Object message, String path);` ===> 请求的资源大于服务器允许的大小。
- `result414(Object message, String path);` ===> 请求的资源URL长于服务器允许的长度。
- `result415(Object message, String path);` ===> 请求资源不支持请求项目格式。
- `result416(Object message, String path);` ===> 请求中包含Range请求头字段，在当前请求资源范围内没有range指示值。
- `result417(Object message, String path);` ===> 服务器不满足请求Expect头字段指定的期望值。
```

### 该类型状态码表示服务器或网关错误。

```markdown
- `result500(Object message, String path);` ===> 服务器错误。
- `result501(Object message, String path);` ===> 服务器不支持请求的功能。
- `result502(Object message, String path);` ===> 网关错误。
- `result503(Object message, String path);` ===> 无法获得服务。
- `result504(Object message, String path);` ===> 网关超时。
- `result505(Object message, String path);` ===> 不支持的http版本。
```

## 如何使用

> 这边只有一个测试模板大家只需要更换数字即可直接使用其他需要的API。

```java
/**
 * Result 实例类
 * @author HCY
 * @since 2021/4/29 8:34 上午
*/
public class ResultExample {
    public static void main(String[] args) {
        Result result = result200();
    }

    /**
     * 其他都类似的
     * 只是修改了Result().result`200`()中的部分引号部分
     * @author HCY
     * @since 2021/4/29 8:42 上午
     * @return com.hcy.code.result.Result
    */
    private static Result result200() {
        return new Result().result200("[返回的具体信息](可以是万物)","[调用Controller的路径](API路径)");
    }
}
```
